## Two-pass methodology

Two complementary audits ran:

1. **Outcome audit** — read engine logs + Postgres tables to
   compute realized P&L, hit rate, hold time per strategy.
2. **Mechanism audit** — read strategy code to ask whether each
   one's underlying thesis can capture real alpha given Kalshi
   fees, microstructure, and implementation details.

The two audits converged on the same verdicts.

## Critical correction: Kalshi fee structure

The first pass used a wrong fee model (~7% flat). The actual
Kalshi taker fee is:

```
fee_cents = ceil(0.07 × N × P × (1 - P))   with a 1¢ floor
```

| Price (¢) | Fee (¢) | % of contract |
|---:|---:|---:|
| 5 | 1 | 20% |
| 10 | 1 | 10% |
| 30 | 2 | 7% |
| 50 | 2 | 4% |
| 70 | 2 | 3% |
| 90 | 1 | 1% |
| 96 | 1 | 1% |

**Implications**:
- Fees are *gentlest near the rails* (1¢ floor) and worst around
  50¢ (2¢ ≈ 4%).
- Penny-priced YES (1-4¢) is structurally fee-fragile: 20-33% in
  fees on the round trip even when the bet wins.
- The `settlement` thesis (buy at 96¢, settle to $1) is viable
  with this fee model — round-trip cost is ~1¢, not the ~7¢
  the wrong model implied.

## Per-strategy verdicts

### KEEP / TIGHTEN — thesis sound, math correct

#### `wx-stat` (highest conviction)

- **Alpha source**: NBM 21-quantile ensemble probabilistic
  forecast → CDF interpolation at the Kalshi-side strike →
  Platt-calibrated `model_p`. Counterparty: weather-market
  participants who don't ingest NBM.
- **Mechanism real?** Yes. NBM is the operational best-skill US
  ensemble for surface temperature; the curator's path
  (`bin/wx-stat-curator/src/nbm_curate.rs`) implements the
  derivation correctly, including same-day observed-extreme gating.
- **Math**: `(bet_p − ask − fee/100) ≥ min_edge_cents`. Half-Kelly
  sizing.
- **Implementation issue (fixed)**: no per-ticker notional cap +
  no min-ask gate meant Kelly could load 20 contracts of YES at
  3¢ on a single ticker. Now capped at $5/ticker
  (`max_notional_per_fire_cents=500`) and gated by `min_ask_cents=5`.
- **Force-flatten bias**: heavy. Many wx-stat YES legs were
  intermediate-priced bets that would have settled fair at $1;
  they were dumped at 1¢. **Discard the realized P&L**.
- **Verdict**: keep, watch ≥30 unforced closed trades.

#### `internal-arb`

- **Alpha source**: per-event-family Σ YES asks < 100¢. Real
  arbitrage when it fires.
- **Math**: with 3 legs at 30¢ each (90¢ total) and ~2¢ fee per
  leg = 6¢ total fees → fires at edge ≥ 2¢ default = locks
  +2-4¢ per package.
- **Constraint**: opportunity frequency, not math. Fired 0 times
  in last 7 days.
- **Verdict**: keep small, harmless if dormant.

#### `implication-arb`

- **Alpha source**: child⊂parent implication; `child_yes_bid −
  parent_yes_ask ≥ edge` locks cash spread upfront. Real arb.
- **Math**: 2-leg fee stack (~2-3¢ at typical prices). Default
  `min_edge_cents=2` covers it.
- **Implementation issue (already fixed pre-audit)**: pair
  refire-loop stacking 7× the same arb on a stale book —
  package-aware scale caps now in place.
- **Verdict**: keep with current per-pair caps.

### MARGINAL — mechanism plausible, fee/calibration-fragile

#### `stat`

- **Alpha source**: Claude Sonnet model_p. Counterparty: anyone
  whose price doesn't match Claude's read.
- **Mechanism issue**: Claude is *not provably calibrated* on
  Kalshi sports/econ priors. Possible systematic over-confidence
  on favorites; possible anchoring on Kalshi's own price hint
  in the prompt.
- **Math**: edge = `model_p × 100 − ask − fee ≥ min_edge_cents`
  is correct. Per-fire EV at 50¢ ask, model_p=0.60: +8¢ per
  contract IF model_p is right.
- **2026-05-08 changes**:
  1. New `min_ask_cents=5` gate (skip penny-priced YES).
  2. All 9 active stat rules disabled (econ markets pending
     recalibration; weather already partitioned to wx-stat).
- **Verdict**: keep on probation; don't re-enable rules until
  there's empirical calibration evidence.
- **2026-05-09 update — paper-trader apparatus shipped.** A
  dedicated evidence-gathering loop now runs every 5 min:
  `predigy-paper-trader record` reads stat-curator output,
  computes the same after-fee edge the live strategy would, and
  writes a `paper_trades` row when it clears threshold;
  `reconcile` scores each row on settlement. Re-enable
  conditions are now mechanical:
  1. `predigy-paper-trader report` shows ≥30 settled trades in
     a given (category, source) bucket.
  2. After-fee EV per trade is positive.
  3. Brier score < 0.25.
  4. Live rules file scoped to that proven category.

#### `settlement`

- **Alpha source**: late-game humans selling early into thin
  ask stacks at 88-96¢.
- **Math**: at 93¢ entry settling to $1, gross +7¢, fee ~1¢, net
  +6¢ if YES wins; lose −94¢ if NO. Break-even hit-rate ~94%.
  Narrow but viable with the actual fee model.
- **A6 profit-lock** (`profit_lock_threshold_cents=5`,
  `profit_lock_min_secs_to_close=180`) already shipped — locks
  the +5¢ favorable move, ducks the close-time race.
- **Verdict**: keep with A6 lock; the 5:1 ratio threshold is a
  heuristic that should be empirically tuned.

#### `cross-arb`

- **Alpha source**: Polymarket mid as a reference price; bet
  Kalshi converges. Stat-arb, not arb.
- **Mechanism issue**: Polymarket gas + USDC frictions mean
  small Poly-vs-Kalshi gaps don't actually arb in reality, so
  some persistent gap is a *real equilibrium* reflecting
  participant-pool differences, not mispricing.
- **2026-05-08 changes**:
  1. `min_edge_cents` default 1 → 3. Fee floor at 30-70¢
     contracts is 1-2¢; firing at 1¢ raw edge was buying a
     fee-loss.
  2. 18 stale `venue-reconcile` rows purged from `positions`
     table — they were stranding the contract cap with phantom
     50¢ entries.
- **Verdict**: keep on probation; watch for actual fires post-
  purge.

### DISABLED — thesis weak / fatal at current state

#### `book-imbalance`

- **Alpha claim**: 0.7 touch imbalance (5.7:1) → fade weak side.
- **Mechanism issue**: displayed touch quantity is well-known
  to be unreliable; Kalshi makers cancel constantly. Live
  observation: bought both YES@85¢ AND NO@15¢ on the same
  ticker → guaranteed fee-loss.
- **2026-05-08 action**: disabled by unsetting
  `PREDIGY_BOOK_IMBALANCE_CONFIG` in `~/.zprofile`.
- **Re-enable condition**: signal-quality study showing
  displayed-imbalance has positive EV after fees on a defined
  market subset, AND a pre-trade gate that blocks same-ticker
  both-side entry.

#### `variance-fade`

- **Alpha claim**: fade 8¢ moves on stable-info markets.
- **Mechanism issue**: no news suppression — fading legitimate
  info trends. No probabilistic edge model.
- **2026-05-08 action**: disabled by unsetting
  `PREDIGY_VARIANCE_FADE_CONFIG`.
- **Re-enable condition**: news-suppression gate that rejects
  fades when a relevant news event has fired in last N min.
  Likely depends on news-trader classifier shipping first.

#### `news-trader`

- **Status**: dormant. Strategy code is fine; classifier service
  doesn't exist.
- **2026-05-08 action**: disabled by unsetting
  `PREDIGY_NEWS_TRADER_ITEMS_FILE`.
- **Re-enable condition**: classifier service running and
  appending to the JSONL file.

#### `latency`

- **Alpha claim**: race the Kalshi book on NWS alerts.
- **Mechanism issue**: structurally infeasible without FIX +
  co-located VPS. Public NWS RSS + REST submit puts us 200-500ms
  behind FIX takers. Tier-3 1¢ force-flat floor turns every
  non-converging long into max-loss.
- **2026-05-08 action**: disabled by unsetting
  `PREDIGY_LATENCY_RULE_FILE`.
- **Re-enable condition**: Phase 4b (FIX access) +
  Phase 4b-VPS (us-east-2) deployed.
